## Defenition
The toggle switch allows users to instantly change an option's state (on/off) [1].
<Canvas of={ToggleSwitchStories.Default} />

## Usage 
### When to use
- Use when users need to change the state of system functionalities and preferences [1].
- Use when a single selection is required and you want to offer two options for **an on/off** type of decision [3].
- Use when it has to **take immediate effect**Â and should not require the user to click Save or Submit to apply the new state [1].
- Use when there is a default value [2].

### When not to use
- No dot use if more actions are needed before the validation, use checkbox instead.
- Do not use if you want to change a contextual view, use a toggle button instead.
- Do not use in a context where an action (save or submit) is required after toggling the button such as in a form. It may confuse the user because they won't know if their change will take immediate effect.
